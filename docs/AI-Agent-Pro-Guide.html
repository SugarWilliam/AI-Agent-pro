AI Agent Pro

部署指南 v8.2.1

项目概述

AI Agent Pro 是一个纯前端应用，基于 HTML/CSS/JavaScript 构建，无需后端服务器即可运行。

项目特点
  • 纯静态文件，部署简单
  • 支持任何静态托管服务
  • 数据存储在浏览器本地
  • 支持PWA离线使用

项目结构

ai-agent-pro/ ├── index.html # 主页面 ├── css/ │ ├── style-new.css # 主样式文件 │ └── chart.css # 图表样式 ├── js/ │ ├── app.js # 应用状态管理 │ ├── ui.js # UI渲染模块 │ ├── events.js # 事件处理 │ ├── llm.js # LLM服务 │ ├── rag.js # RAG服务 │ ├── sync.js # 同步服务 │ └── chart-render.js # 图表渲染 ├── docs/ │ ├── AI-Agent-Pro-Design.html # 设计文档 │ ├── AI-Agent-Pro-Guide.html # 部署指南 │ └── AI-Agent-Pro-Features.html # 功能说明书(H5) └── manifest.json # PWA配置

环境要求

项目要求
Web服务器任何支持静态文件的Web服务器
浏览器Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
网络需要访问外部AI API（可选）
存储空间浏览器LocalStorage（约5-10MB）

部署方式

方式一：本地运行（开发）

1

下载源码

从侧边栏点击"下载源码"按钮，或从GitHub下载。

# 解压到本地目录 unzip ai-agent-pro-source.zip -d ai-agent-pro
2

启动本地服务器

# 使用 Python 3 cd ai-agent-pro python -m http.server 8080 # 或使用 Node.js npx serve . # 或使用 VS Code Live Server 插件
3

访问应用

打开浏览器访问 http://localhost:8080

方式二：Nginx部署（生产）

1

安装Nginx

# Ubuntu/Debian sudo apt update sudo apt install nginx # CentOS/RHEL sudo yum install nginx # macOS brew install nginx
2

配置Nginx

# 创建配置文件 sudo nano /etc/nginx/sites-available/ai-agent-pro # 添加以下内容 server { listen 80; server_name your-domain.com; root /var/www/ai-agent-pro; index index.html; location / { try_files $uri $uri/ /index.html; } # 启用gzip压缩 gzip on; gzip_types text/css application/javascript; }
3

部署文件

# 复制项目文件 sudo cp -r ai-agent-pro/* /var/www/ai-agent-pro/ # 设置权限 sudo chown -R www-data:www-data /var/www/ai-agent-pro # 启用站点 sudo ln -s /etc/nginx/sites-available/ai-agent-pro /etc/nginx/sites-enabled/ sudo nginx -t sudo systemctl restart nginx

方式三：Docker部署

1

创建Dockerfile

FROM nginx:alpine COPY . /usr/share/nginx/html EXPOSE 80
2

构建并运行

# 构建镜像 docker build -t ai-agent-pro . # 运行容器 docker run -d -p 8080:80 --name ai-agent-pro ai-agent-pro

方式四：Cloudflare Pages

1

上传代码

将代码推送到GitHub仓库

2

连接Cloudflare

  1. 登录 Cloudflare Dashboard
  2. 点击 "Pages" → "Create a project"
  3. 连接GitHub仓库
  4. 设置构建命令：留空（纯静态）
  5. 设置输出目录：/
  6. 点击 "Save and Deploy"

方式五：GitHub Pages

1

配置GitHub Pages

  1. 将代码推送到GitHub仓库
  2. 进入仓库 Settings → Pages
  3. Source 选择 "Deploy from a branch"
  4. Branch 选择 "main"，文件夹选择 "/ (root)"
  5. 点击 Save
部署完成后，访问 https://your-username.github.io/repo-name

配置说明

API Key配置

应用内置了部分API Key，但建议配置自己的Key：

  1. 打开应用，进入设置 → 模型
  2. 点击对应模型的 "配置" 按钮
  3. 输入你的API Key
  4. 点击保存
安全提示

API Key存储在浏览器本地存储中，请确保使用安全的网络环境。

故障排除

常见问题

问题解决方案
页面空白检查浏览器控制台是否有JS错误，确保使用现代浏览器
API调用失败检查API Key是否正确，网络是否可访问API端点
数据丢失检查浏览器LocalStorage是否被清除，定期导出备份
样式错乱清除浏览器缓存，强制刷新页面 (Ctrl+F5)
